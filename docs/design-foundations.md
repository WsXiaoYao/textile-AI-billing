# 设计基础规范

整理日期：2026-04-27

这份文档定义“纺织开单小工具”的基础视觉语言。后续页面开发优先复用这里的颜色、字号、间距、圆角、阴影、按钮、标签和图标规则。

对应全局样式文件：`styles/foundations.wxss`

## 1. 设计方向

产品气质：微信原生感 + 工具型 SaaS + 销售开单效率。

页面应该安静、清晰、可扫读，不做营销页式大装饰。蓝色用于行动和选中，白色卡片承载业务对象，灰色组织层级，红绿黄只表达状态。

## 2. 颜色

基础色：

- 页面背景：`#F6F7F9`
- 卡片/面板：`#FFFFFF`
- 轻底色：`#F8FAFC`
- 分割线：`#EDF1F5`
- 边框：`#E3E8EF`

文字色：

- 主文本：`#172033`
- 次文本：`#667085`
- 弱文本：`#98A2B3`
- 反白文本：`#FFFFFF`

品牌与行动：

- 主蓝：`#1677FF`
- 主蓝按下：`#0F5ED7`
- 蓝色浅底：`#EAF3FF`
- 蓝色边框：`#B9D7FF`

状态色：

- 成功绿：`#12B76A`
- 成功浅底：`#EAF8F0`
- 警告黄：`#F79009`
- 警告浅底：`#FFF6E5`
- 错误红：`#E5484D`
- 错误浅底：`#FFECEE`

业务使用规则：

- 蓝色只用于主按钮、链接、选中态、重点入口。
- 红色只用于欠款、错误、删除、危险操作，不做普通装饰。
- 绿色只用于识别成功、可继续、已加入。
- 黄色只用于待确认、部分识别、需补充。
- 页面不做大面积渐变，不让蓝色覆盖所有模块。

## 3. 字体

使用系统字体：

```css
-apple-system, BlinkMacSystemFont, "PingFang SC", "Helvetica Neue", Arial, sans-serif
```

字号层级：

- 页面标题：`36rpx`，700，仅用于页面主标题。
- 模块标题：`32rpx`，700，用于卡片或分区标题。
- 正文：`28rpx`，400/500，用于主要内容。
- 辅助文字：`24rpx`，400，用于说明、标签、时间。
- 金额重点：`44rpx`，700，用于应收、合计、欠款。

排版规则：

- 不使用负字距。
- 金额、客户名、商品名优先保证可读。
- 长客户名可换行或省略，但不能压住金额。
- 商品规格过长时换行，不挤掉数量和金额。

## 4. 间距

基础间距使用 8rpx 倍数：

- `8rpx`：紧密元素间距。
- `16rpx`：按钮/标签内部或相邻控件间距。
- `24rpx`：卡片内部小间距。
- `32rpx`：页面左右边距、卡片内边距。
- `48rpx`：大模块上下间距。

页面规则：

- 手机页面左右边距默认 `32rpx`。
- 卡片内边距默认 `24rpx` 到 `32rpx`。
- 列表行高度不低于 `96rpx`。
- 底部输入区要考虑安全区，使用 `padding-bottom: env(safe-area-inset-bottom)`。

## 5. 圆角、边框、阴影

圆角：

- 小控件：`8rpx`
- 标签/输入框：`12rpx`
- 卡片：`16rpx`，对应 8px，不超过此前约定。
- 胶囊按钮：使用 `999rpx`

边框：

- 默认边框：`1rpx solid #E3E8EF`
- 分割线：`1rpx solid #EDF1F5`

阴影：

- 默认卡片阴影：`0 8rpx 24rpx rgba(23, 32, 51, 0.06)`
- 浮层阴影：`0 16rpx 48rpx rgba(23, 32, 51, 0.12)`
- 不给每个模块都加阴影，只给真实卡片、底部操作区、浮层使用。

## 6. 按钮

高度：

- 主要按钮：不低于 `80rpx`
- 次按钮：不低于 `72rpx`
- 图标按钮：不低于 `64rpx`

按钮类型：

- 主按钮：蓝底白字，一页原则上只有一个。
- 次按钮：白底蓝字或白底深色字。
- 危险按钮：红色文字或红色浅底，不默认做大红实心。
- 文本按钮：用于编辑、更多、查看详情。

按钮文案：

- 用动作描述，例如“识别”“提交订单”“清空购物车”。
- 避免只写“确定”“确认”这类模糊文案。

## 7. 输入与表单

输入框：

- 高度默认 `88rpx`。
- 背景 `#FFFFFF` 或 `#F8FAFC`。
- 边框 `#E3E8EF`。
- 聚焦态用主蓝边框。
- 错误态用红色边框和红色提示。

开单输入：

- 底部输入区固定在页面底部时，必须给内容区留出底部空间。
- 输入区主按钮用“识别”，不要让多个主按钮并列。
- 识别失败在输入区附近反馈，不跳页。

## 8. 卡片与列表

卡片用于真实业务对象：

- 客户卡。
- 订单卡。
- 商品/购物车项。
- 金额汇总。

页面分区不要全部卡片化。工具栏、标题区、筛选区可以保持无框或轻底色。

列表规则：

- 列表行主信息在左，状态/金额在右。
- 金额列宽度要稳定。
- 行可点击时整行可点。
- 行内危险操作不要和主操作贴太近。

## 9. 标签与状态

标签高度：

- 小标签：`40rpx`
- 普通标签：`48rpx`

标签颜色：

- 客户标签：蓝色浅底。
- 成功状态：绿色浅底。
- 待确认：黄色浅底。
- 欠款/错误：红色浅底。
- 普通分类：灰色浅底。

状态文案：

- 使用“已识别”“待确认”“识别失败”“已加入”“有欠款”。
- 不使用只有颜色、没有文字的状态。

## 10. 图标

图标来源：Lucide Icons。

尺寸：

- 小图标：`32rpx`
- 常规图标：`40rpx`
- 工具按钮图标：`44rpx`
- tabBar PNG：`81px * 81px`

规则：

- 页面内普通图标优先使用本地 SVG。
- tabBar 使用已生成的本地 PNG。
- 重要操作不能只有图标，必须有文字或明确上下文。
- 同一页面不要混用多套图标风格。

## 11. 常用基础类

已在 `styles/foundations.wxss` 中准备：

- 页面：`.app-page`
- 卡片：`.app-card`
- 分区标题：`.app-section-title`
- 主/次/危险按钮：`.btn`、`.btn-primary`、`.btn-secondary`、`.btn-danger`
- 标签：`.tag`、`.tag-primary`、`.tag-success`、`.tag-warning`、`.tag-danger`
- 文本：`.text-primary`、`.text-secondary`、`.text-muted`
- 金额：`.amount-strong`
- 图标：`.icon-sm`、`.icon-md`、`.icon-lg`

## 12. 小程序组件宽度

自定义组件必须显式声明宿主节点尺寸。默认建议：

```css
:host {
  display: block;
  width: 100%;
}
```

页面负责左右边距，例如 `padding: 0 24rpx` 或 `32rpx`；组件内部根节点使用 `width: 100%`。不要在组件内部写死 `702rpx`、`750rpx` 这类整屏宽度，否则在真实小程序组件宿主、不同设备宽度、父级 flex 场景中容易收缩、溢出或错位。

Figma `375px` 画板转小程序时，先把系统 UI 剥离，再把真实业务区域转成可伸缩布局。

## 13. 验收清单

每个页面开发完成后检查：

1. 是否使用了本规范里的颜色，而不是临时取色。
2. 主按钮是否只有一个，且高度不低于 `80rpx`。
3. 客户名、商品名、金额是否不会互相挤压。
4. 卡片圆角是否不超过 `16rpx`。
5. 页面是否大面积单一蓝色或强渐变。
6. 错误、成功、待确认状态是否有文字说明。
7. 图标是否来自统一来源。
8. 小屏幕下点击区域是否足够大。
9. 自定义组件是否有 `:host`，并且卡片没有变窄、文字没有竖排、金额没有被截断。
