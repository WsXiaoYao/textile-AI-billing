# 聚云掌柜小程序设计规范

本文档用于约束后续新页面、组件和局部改版的视觉与交互风格。目标不是做营销型页面，而是保持一个适合纺织开单、客户往来、库存和收款场景的高效业务工具。

## 设计定位

聚云掌柜是中文微信小程序形态的经营工具。界面应当安静、清晰、可扫描，优先服务重复操作和快速核对。

不要做大面积宣传式 hero、装饰性插画、渐变光斑、复杂动效或过度卡片化包装。页面第一屏应直接进入业务操作或数据列表。

## 基础画布

- 设计稿基准：`375px` 宽，对应小程序 `750rpx`。
- 换算：`1px = 2rpx`。
- 页面背景：`#f4f7fb` 或全局 `#f6f7f9`。
- 内容横向安全边距：常规 `24rpx`。
- 底部固定操作区必须考虑 `env(safe-area-inset-bottom)`。
- 可点击区域建议不小于 `88rpx` 高或宽，约等于 44px 触控目标。

## 色彩

主色用于主要按钮、激活态、链接、当前页签和关键可操作入口。

```text
Primary Blue:     #2E7BEF / #2F7DF6
Text Strong:      #172033 / #1d2129
Text Secondary:   #667085
Text Muted:       #98a2b3
Page Background:  #f4f7fb / #f6f7f9
Card Background:  #ffffff
Border:           #d8e5f6
Success Green:    #0e8a62
Danger Red:       #d94a55
Warning Orange:   #d77a00 / #f79009
Info Cyan:         #0794b8
```

金额颜色规则：

- 未收、欠款、异常：红色 `#d94a55`。
- 已收、结清、成功：绿色 `#0e8a62`。
- 预收、主操作、当前选择：蓝色 `#2E7BEF`。
- 普通金额和说明：深色正文或灰色说明。

## 字体

全局字体使用系统中文字体：

```css
font-family: -apple-system, BlinkMacSystemFont, "PingFang SC", "Helvetica Neue", Arial, sans-serif;
```

字号建议：

- 页面/弹窗大标题：`32-36rpx`，`font-weight: 900`。
- 卡片主标题：`30-32rpx`，`font-weight: 800/900`。
- 正文和表单值：`26-28rpx`。
- 辅助说明、元信息：`22-24rpx`。
- 状态标签：`20-22rpx`，加粗。
- 大金额：`34-44rpx`，加粗。

不要用 viewport 宽度动态缩放字体。中文文案要避免孤字、寡行和截断后语义不完整。

## 布局

页面结构优先采用：

```text
搜索/筛选工具区
汇总卡片
业务列表或表单
底部固定操作区
```

常用间距：

- 页面内边距：`24rpx`。
- 卡片内边距：`22-28rpx`。
- 卡片间距：`18-24rpx`。
- 表单字段高度：`96-112rpx`。
- 工具按钮高度：`68-78rpx`。
- 标签高度：`30-48rpx`。

不要把卡片套在卡片里。页面区域应是全宽布局或轻量分区，卡片只用于独立业务对象、表单组、弹窗和列表项。

## 卡片

常规业务卡片：

```css
border: 1rpx solid #d8e5f6;
border-radius: 20-24rpx;
background: #ffffff;
box-shadow: 0 8rpx 24rpx rgba(13, 31, 61, 0.05);
```

列表卡片应信息密度高，主内容一眼可扫：

- 左侧放名称、编号、电话、地址/摘要。
- 右侧放状态和核心金额。
- 标签只放必要信息，避免单独占行。
- 相同语义不要重复展示，例如手机号不要在 meta 和标签里各出现一次。

## 按钮

主要按钮：

```css
background: #2e7bef;
color: #ffffff;
border-radius: 14-16rpx;
font-weight: 900;
```

次要按钮：

```css
border: 1rpx solid #b9d7ff;
background: #f2f7ff;
color: #2e7bef;
```

危险按钮只在明确破坏性动作中使用红色。普通返回、取消、筛选重置不要用红色。

图标按钮优先使用 Lucide 风格的线性图标，保持同一套图标体系。不要混用临时手绘图标、不同线宽或不同风格的图标。

## 表单

表单适合用白色表单卡承载，字段之间用浅分割线。标签放上方或左侧，值文本要比标签更醒目。

表单字段规则：

- 必填校验在提交时明确提示。
- 输入框 placeholder 使用 `#98a2b3`。
- picker 行显示当前值，未选择时显示浅色提示。
- 金额字段统一保留两位小数。
- 长地址、备注使用多行输入，不要强行单行截断。

## 筛选和列表

筛选区域应固定在列表上方，支持快速扫读：

- 状态 tab：全部、有欠款、已结清、有预收、最近下单。
- 排序、筛选、日期类入口用紧凑按钮。
- 分类筛选必须来自后端字典，不要写死数组。
- 搜索框 placeholder 用中文，比如“搜索客户、电话、地区”。

列表空态要短，不写功能说明书式长文案。示例：

```text
没有匹配的客户
换个客户名、电话或筛选条件再试试。
```

## 状态标签

状态标签使用固定高度、居中布局：

```css
height: 34-48rpx;
border-radius: 999rpx;
display: flex;
align-items: center;
justify-content: center;
font-size: 20-22rpx;
font-weight: 800/900;
```

标签语义：

- `已结清`：绿色。
- `有欠款`：红色。
- `有预收`：蓝色。
- `待处理/待上传`：橙色。
- `停用/无数据`：灰色。

短文本必须真居中，不靠手动微调。

## 弹窗和底部面板

底部弹窗用于编辑、筛选、二次确认。样式：

- 遮罩：`rgba(15, 23, 42, 0.38)`。
- 面板：顶部圆角 `28rpx 28rpx 0 0`。
- 顶部 handle：`72rpx x 8rpx`。
- 底部按钮区考虑安全区。

弹窗内不要塞过多解释文字。需要说明时用一条浅蓝提示条。

## 导航

底部 tab 保持五项：

```text
首页 / 订单 / 客户 / 更多 / 我的
```

“更多”承载低频管理入口，例如产品分类、客户分类、仓库、供应商、采购、报表。新增管理页优先从“更多”进入。

微信右上角胶囊按钮是系统区域，不把页面内容放到它下面，也不把它当装饰元素重画。

## 业务页面约定

客户相关：

- 客户分类来自 `customer_categories`，不是写死数组。
- 客户卡片右上标签表示客户分类。
- 地区使用 `province/city/district`，不要和客户分类混用。
- 电话只展示一次。
- 客户详情应分为金额概览、销售记录、资金流水、基础资料。

金额往来：

- 未收、欠款、退款风险用红色。
- 已收、结清用绿色。
- 预收用蓝色。
- 收款分摊、资金流水要能追溯到单据。

导入导出：

- 模板、上传、导出状态要明确。
- 失败原因要能被用户看懂。
- 任务列表中状态文案不要使用英文。

## 文案

所有导航、按钮、状态、提示、空态默认使用简体中文。

文案原则：

- 短、具体、像业务工具。
- 不写营销语。
- 不用英文状态词。
- 不在页面内解释“如何使用这个页面”的长段文字。

推荐：

```text
新增客户
保存
筛选
已结清
有欠款
没有匹配的客户
```

避免：

```text
Start
Submit
Customer Management
本页面支持用户根据各种复杂条件进行客户数据查询与维护操作
```

## 实现习惯

- 优先复用现有页面结构、色彩和组件样式。
- 新页面的 `json` 导航栏背景使用 `#2E7BEF` 或 `#2F7DF6`。
- 新页面背景使用 `#f4f7fb`。
- 固定底部按钮使用 `calc(... + env(safe-area-inset-bottom))`。
- 表格类数据在小程序里优先转成卡片列表，不做横向难读表格。
- 页面加载失败用 toast；空结果用页面空态。
- 后端真实数据字段和前端展示字段要明确映射，不为了展示随意新增业务含义不清的字段。

## 检查清单

提交新页面或新组件前检查：

- 中文是否截断、孤字、寡行。
- 按钮和状态标签是否水平/垂直居中。
- 同一信息是否重复展示。
- 客户分类、地区、状态、金额是否语义分清。
- 金额颜色是否符合规则。
- 触控区域是否足够大。
- 底部固定区是否避开安全区。
- 图标是否来自同一套体系。
- 页面是否像业务工具，而不是营销落地页。
